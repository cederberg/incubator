# Review Checklist: Other

Apply only the sections that match the requested document type or the draft's
evident purpose.

## Baseline

- [ ] Audience and purpose are clear from the opening section
- [ ] Scope is clear: the reader can tell what the document covers and omits
- [ ] Structure matches the document type and does not force feature/process
      sections where they do not fit
- [ ] No section heading contains a placeholder such as `{{Topic}}`
- [ ] Domain terms are consistent throughout
- [ ] Implementation detail appears only when it supports the document purpose

## Coverage

- [ ] Relevant actors, systems, components, or source materials are named
- [ ] Claims are supported by research findings
- [ ] Decisions state the condition, options, and outcome
- [ ] Constraints, risks, and dependencies are explicit when relevant
- [ ] Outcomes, acceptance criteria, or recommendations are stated when relevant
- [ ] Open questions are separated from established facts

## Process or Behaviour

- [ ] The trigger is stated
- [ ] Steps appear in execution order
- [ ] Decision points state each possible path
- [ ] Failure behaviour is documented inline or in a dedicated section

## Requirement or Change

- [ ] Requirement statements are clear and testable
- [ ] Current and proposed behaviour are distinguished
- [ ] Acceptance criteria cover success, failure, and important edge cases
- [ ] User-visible, operational, and integration impacts are covered where
      relevant
