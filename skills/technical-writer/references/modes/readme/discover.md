# Discovery and Research: README

## Phase 1: Discovery

Use the default discovery instructions.

## Phase 2: Research Topics

| Topic | Description | Output file |
|---|---|---|
| Purpose & audience | What the project does, for whom, and in what context; any notable design constraints or goals visible in the code or documentation | purpose.md |
| Setup & prerequisites | What a user needs before they can use the project; how to install it; any runtime dependencies or platform requirements | setup.md |
| Usage | The primary commands, invocations, or API entry points; key options or parameters and their effects; representative input/output pairs | usage.md |
| Configuration | Named configuration options, their defaults, and their effects on behaviour; where configuration lives (file, environment variable, flag) | config.md |
| Contribution & support | How to build from source, how to run tests, how to contribute, where to get help or file issues | contributing.md |

Adapt scopes to the actual project using `$WORK_DIR/discovery.md`. For a small project, use two or three researchers.
