# Mode Instructions: README

## Research Topics

| Topic | Description | Output file |
|---|---|---|
| Purpose & audience | Cover what the project does, for whom, and in what context. Note any design constraints or goals visible in the code or documentation. | purpose.md |
| Setup & prerequisites | Cover what the user needs before using the project. Include installation steps, runtime dependencies, and platform requirements. | setup.md |
| Usage | Cover primary commands, invocations, or API entry points. Include key options or parameters, their effects, and representative input/output pairs. | usage.md |
| Configuration | Cover named configuration options, their defaults, and their effects on behaviour. State where configuration lives (file, environment variable, or flag). | config.md |
| Contribution & support | Cover how to build from source, run tests, and contribute. Include where to get help or file issues. | contributing.md |

Adapt scopes to the actual project using `$WORK_DIR/discovery.md`. For a small project, use two or three researchers.
