# Mode Instructions: README

## Research Topics

Research each topic and write its findings to the named output file:

- **Purpose & audience** → `purpose.md` \
  Cover what the project does, for whom, and in what context. Note any design
  constraints or goals visible in the code or documentation.
- **Setup & prerequisites** → `setup.md` \
  Cover what the user needs before using the project. Include installation
  steps, runtime dependencies, and platform requirements.
- **Usage** → `usage.md` \
  Cover primary commands, invocations, or API entry points. Include key options
  or parameters, their effects, and representative input/output pairs.
- **Configuration** → `config.md` \
  Cover named configuration options, their defaults, and their effects on
  behaviour. State where configuration lives (file, environment variable, or
  flag).
- **Contribution & support** → `contributing.md` \
  Cover how to build from source, run tests, and contribute. Include where to
  get help or file issues.

Adapt scopes to the actual project using `$WORK_DIR/discovery.md`. For a small
project, use two or three researchers.
